# AppLocker 产品问题清单

> 更新：2026-06-14
> 目标：以终为始，做值得用户付费的产品

---

## 产品价值评估（以终为始）

### 核心问题：用户会花 8 元买这个产品吗？

**当前状态：⚠️ 接近值得购买，但缺少"惊喜感"**

| 维度 | 评分 | 说明 |
|------|------|------|
| 核心功能可靠性 | 🟡 接近 OK | 需要真机验证 |
| 差异化（定时专注） | 🟢 独特 | 竞品均不支持此功能 |
| 操作便捷性 | 🟢 3步完成 | 远优于 Screen Time (7步) |
| 数据统计 | 🟢 完整 | 图表、连胜、CSV导出 |
| 安全保护 | 🟢 SHA256+FaceID | 无法绕过 |
| **惊喜感/仪式感** | **🔴 缺失** | **无Widget、无Siri、无分享、无动画** |
| **功能完整度** | **🟡 可用但不够** | **无预设、无后台任务、无快捷操作** |

### 用户痛点分析

**目标用户：** 控制不住自己频繁刷手机 / 需要专注工作/学习的年轻人

**核心痛点：** "我知道应该放下手机，但我控制不住自己"

**现有方案问题：**
1. iOS Screen Time：设置复杂(7步+)、无定时功能、可以绕过
2. Forest App：¥12，但不真的屏蔽App，只是种树激励
3. Opal/Focusmate：订阅制(¥30+/月)，太贵
4. 应用锁Pro：¥6，但只防别人看，不解决自我控制

**我们的差异化：** 定时屏蔽 + 游戏化激励 + 一键操作 = 真正帮用户"做到"

### 付费决策的三要素

1. **核心可靠** — 锁定真的有效、定时真的解锁 🔴 **基本满足**
2. **差异化足够** — 定时专注模式是独特优势 🟢 **满足**
3. **有惊喜感** — 用户打开就觉得值这个价 🔴 **严重缺失 ← 本轮重点**

## P0 - 核心差异化缺失（影响付费意愿）

### [P0-1] 缺少 Widget 小组件 🔴
- **问题**：用户想快速开始专注必须打开 App → 主页 → 点击开始，流程太长
- **影响**：这是最重要的一键入口，没有 Widget 用户使用频率会大幅下降
- **修复方向**：WidgetKit 小组件（桌面 + 锁屏），支持：
  - 一键"开始专注25分钟"
  - 显示当前专注状态+倒计时
  - 显示今日专注统计
- **工作量**：~2小时
- **参考**：StartFocus Pro 的 Widget

### [P0-2] 缺少 Siri 快捷指令 🔴
- **问题**：用户无法通过语音快速开始/结束专注
- **影响**：专注场景通常发生在用户已放下手机时（开始学习/工作），此时需要最轻量的启动方式
- **修复方向**：Intents Extension，支持：
  - "Hey Siri, 开始专注25分钟"
  - "Hey Siri, 结束专注"
- **工作量**：~1.5小时

### [P0-3] 缺少专注预设（Presets）🔴
- **问题**：每次都要重新选择要屏蔽的应用，没有常用配置
- **影响**：用户有不同场景（工作时间、学习时间、睡前），每次重新选应用很麻烦
- **修复方向**：添加预设系统（工作/学习/个人），可保存多个 App 选择方案
- **工作量**：~1.5小时

### [P0-4] 缺少仪式感 / 成就感 🟡
- **问题**：专注开始和结束没有任何动画/音效反馈，体验平淡
- **影响**：用户需要"仪式感"来建立正反馈循环
- **修复方向**：
  - 开始专注：锁屏动画 + 简短音效
  - 结束专注：庆祝动画 + 成就通知
  - 每日目标达成：推送通知
- **工作量**：~1小时

## P1 - 功能/体验问题

### [P1-1] FamilyActivitySelection 未持久化 🟡
- **问题**：`FamilyActivitySelection` 不遵循 Codable，选择的 App 列表在 App 重启后丢失
- **影响**：用户每次打开 App 都需要重新选择要屏蔽的应用
- **修复方向**：存储 token 名称列表（`applicationTokens` -> 名称数组），重启后重新加载
- **工作量**：~0.5小时
- **位置**：`ShieldManager.swift`

### [P1-2] 后台定时器不准确 🟡
- **问题**：`Timer.scheduledTimer` 在 App 进入后台后暂停，虽然加了本地通知和前台唤醒检查，但仍有窗口期
- **影响**：用户预期的时间到了就自动解锁，但实际上可能延迟（直到下次打开 App）
- **修复方向**：使用 BGTaskScheduler 注册后台任务
- **工作量**：~1小时
- **位置**：`LockStore.swift`

### [P1-3] 授权拒绝后引导不足 🟡
- **问题**：如果用户拒绝了屏幕使用时间授权，主页没有任何提示，用户不知道 App 为什么不能工作
- **影响**：用户安装后看不到核心功能，直接删除
- **修复方向**：主页检查授权状态，未授权时显示引导卡片
- **工作量**：~0.5小时
- **位置**：`HomeView.swift`

### [P1-4] 预览截图显示本地化 key 而不是实际文案 🟡
- **问题**：某些界面使用了 `Text(LocalizedStringKey(...))` 但对应的 key 在 Localizable.strings 中不存在
- **影响**：用户看到 key 名称而不是文案（如 "guide_welcome_title" 而不是 "欢迎来到专注模式"）
- **修复方向**：检查所有 LocalizedStringKey 的 key 是否都存在于 string 文件中
- **工作量**：~0.5小时

### [P1-5] 无专注历史时间线视图 🟡
- **问题**：统计页面只显示聚合数据，没有每日/每周的时间线视图
- **影响**：用户想回顾"我今天什么时候在专注"做不到
- **修复方向**：添加时间线视图，显示当天的专注记录时间轴
- **工作量**：~1小时

### [P1-6] 密码设置体验不佳 🟡
- **问题**：
  - 密码输入没有长度限制/提示
  - 没有密码强度指示
  - 修改密码需要输入旧密码验证（当前没有）
- **影响**：安全性和用户体验都受影响
- **修复方向**：添加旧密码验证 + 密码长度提示
- **工作量**：~0.5小时

## P2 - 代码质量/优化问题

### [P2-1] StatsView 日期标签不国际化
- **问题**：`date_weekday_format` 在 zh 下使用中文字符串格式，但 dayLabel 方法中固定了中文星期数组
- **影响**：中文功能正常，但英文下星期几仍然显示中文
- **位置**：`StatsView.swift` 第 346 行
- **工作量**：~0.2小时

### [P2-2] LockStore 历史记录可能无限增长
- **问题**：没有任何清理机制，历史记录越多 JSON 文件越大
- **影响**：使用一年后可能几百 KB，影响启动速度
- **修复方向**：设置最大历史记录数（例如 1000 条），自动清理旧数据
- **工作量**：~0.3小时
- **位置**：`LockStore.swift`

### [P2-3] HomeView 自定义时长 Sheet 中 TextField 无限输入
- **问题**：用户可以输入 99999 分钟等不合理的数值
- **影响**：1-480 分钟的限制只在确认时校验，输入时无约束
- **修复方向**：添加输入过滤或使用更严格的验证
- **工作量**：~0.2小时

### [P2-4] ContentView 锁定模式下 TabBar 隐藏方式不优雅
- **问题**：目前使用 ZStack + zIndex 来隐藏 TabBar，但这会导致界面闪烁
- **影响**：某些情况下锁定/解锁时界面会闪烁
- **修复方向**：使用 `.fullScreenCover` 展示锁定界面
- **工作量**：~0.3小时

### [P2-5] 没有 App Icon 变体
- **问题**：使用默认 SF Symbol 作为图标
- **影响**：桌面图标没有辨识度
- **工作量**：~0.5小时（需要设计资源）
